# TDD Feature Skill v1

## Metadata

- **name**: tdd-feature
- **description**: Execute Test-Driven Development workflow following the 3-commit pattern (RED → GREEN → REFACTOR)
- **version**: 1.0.0

## Instructions

When the user invokes `/tdd-feature <scope> <description>`, follow this workflow:

### Phase 1: Understanding (Auto-detect)

1. Parse the scope (e.g., auth, scan, care, bouquet)
2. Understand the feature description
3. Identify relevant files in the codebase:
   - Check `src/app/` for route handlers
   - Check `src/lib/` for utilities
   - Look for existing test patterns

### Phase 2: RED Commit (Test First)

1. Write a failing test that describes the expected behavior
2. Test location rules:
   - API routes: `src/app/api/<scope>/<action>/route.test.ts`
   - Components: `src/app/<path>/page.test.tsx` or `src/components/<name>.test.tsx`
   - Utilities: `src/lib/<name>.test.ts`
3. Follow existing test patterns (Vitest, React Testing Library)
4. Import the function/component being tested (it will fail since it doesn't exist)
5. Run `npm test` to confirm test fails
6. Commit with format: `test(<scope>): [RED] <description>`

### Phase 3: GREEN Commit (Minimum Implementation)

1. Write minimum code to make the test pass
2. No refactoring yet - just make it work
3. Run `npm test` to confirm test passes
4. Commit with format: `feat(<scope>): [GREEN] <description>`

### Phase 4: REFACTOR Commit (Clean Up)

1. Review the code for improvements:
   - Remove duplication
   - Improve variable names
   - Add error handling
   - Ensure type safety
2. Run `npm test` to ensure tests still pass
3. Run `npm run lint` to check code style
4. Commit with format: `refactor(<scope>): <what was improved>`

### Phase 5: Verification

1. Run full test suite: `npm test`
2. Run linting: `npm run lint`
3. Report results to user

## Constraints

- NEVER mix test and implementation in the same commit
- ALWAYS follow the 3-commit pattern strictly
- Use existing patterns from similar files in the codebase
- NEVER use `any` types - use `unknown` and narrow
- ALWAYS scope DB queries with `user_id = auth.uid()` for IDOR protection
- Run tests before each commit to verify state

## Example Usage

```
/tdd-feature auth "add password validation to signup"
/tdd-feature scan "implement rate limiting"
/tdd-feature bouquet "add health calculation"
```

## Expected Output

- Three commits following the pattern
- Passing tests
- Clean, refactored code
- Session log entry documenting the workflow
